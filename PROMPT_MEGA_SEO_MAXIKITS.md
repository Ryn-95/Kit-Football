## INSTRUCTIONS D'IMPLÉMENTATION

### Pour chaque page, tu dois :

1. **Créer l'entrée de données** dans le fichier `longtail-data.ts` ou `seo-data.ts`
2. **Chaque page doit avoir** :
   - Un `slug` optimisé
   - Un `title` (H1) contenant le mot-clé principal
   - Un `metaTitle` optimisé (<60 chars)
   - Une `metaDescription` optimisée (<155 chars)
   - Un `heroText` engageant
   - Un `seoLongText` (300+ mots de contenu unique)
   - 2-3 `faq` en schema markup
   - 4+ `relatedSlugs` pour le maillage interne
3. **Le contenu doit être unique pour chaque page** — pas de duplicate content
4. **Le maillot interne doit être dense** — chaque page lie vers 4-5 autres pages

### Architecture cible :
- **Pages pilier (silo)** : clubs majeurs, sélections majeures, types de maillots
- **Pages longue traîne** : variantes club × type × saison × audience × rétro
- **Pages rétro** : chaque maillot rétro avec année spécifique
- **Money pages** : pages d'intention d'achat (pas cher, fiable, comparatif)
- **Pages de niche** : concept, blue lock, thailande, ensemble, etc.

### Priorité d'implémentation :
1. 🔴 Pages génériques haute valeur (gros volumes)
2. 🟠 Clubs et sélections manquants (Portugal, Maroc, Italie, etc.)
3. 🟡 Pages rétro manquantes
4. 🟢 Niches spécifiques
